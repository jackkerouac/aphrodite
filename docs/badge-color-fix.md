# Badge Background Color Fix

## Issue
Badge background colors weren't being consistently applied between the Preview page in the frontend and the final output generated by the backend. Specifically:

- When `use_brand_colors` was set to `true` in the database, the brand colors weren't being correctly applied if there was also a background color value stored.
- The color priority logic wasn't handling the `use_brand_colors` setting correctly.

## Changes Made

1. Modified `canvasUtils.js` to change the background color priority order:
   - **Before**: `custom backgroundColor > custom background_color > brand color > default color`
   - **After**: `brand color (when use_brand_colors is true) > custom backgroundColor > custom background_color > default color`

2. Fixed the conditional logic in the `applyBackground` function to ensure that brand colors are always applied when `use_brand_colors` is `true`, regardless of whether there's a stored background color.

3. Added more detailed logging to help diagnose color application issues in the future.

## Expected Behavior

With this fix, badges should now display with consistent colors:

- **Audio badges**: Will appear BLUE (#2E51A2) when `use_brand_colors` is enabled, regardless of the stored background color.
- **Resolution badges**: Will appear RED (#FA320A) when `use_brand_colors` is enabled.
- **Custom colors**: Will still be honored when `use_brand_colors` is disabled.

## Testing

A dedicated test script (`test-badge-colors.js`) was created to verify the fix. The test:

1. Sets up badge configurations with different color scenarios
2. Runs the badge renderer with these configurations
3. Verifies that the brand colors are correctly applied when `use_brand_colors` is enabled

## Potential Follow-up

The frontend Preview page does not explicitly show or toggle the `use_brand_colors` option. This might cause confusion for users if the colors they see in Preview differ from the final result. A future enhancement could be to make this option visible in the Preview interface.
